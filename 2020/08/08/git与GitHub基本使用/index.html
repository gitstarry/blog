#安装

Windows上安装Git：直接在官网上下载，按默认选项安装即可。

#全局配置

git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。

#创建版本库

新建一个目录，在这个目录下打开git bush ，执行git init，会出现一个.git目录，这个目录是Git来跟踪管理版本库的，不要轻易修改，以免破环仓库，同时，注意下这个目录是默认隐藏的。

#只能跟踪文本文件

所有的版本控制系统，其实只能跟踪文本文件的改动，比如TXT文件，网页，所有的程序代码等等。而图片、视频这些二进制文件，虽然也能由版本控制系统管理，但没法跟踪文件的变化，只能把二进制文件每次改动串起来，也就是只知道图片从100KB改成了120KB，但到底改了啥，版本控制系统不知道。Microsoft的Word格式是二进制格式

#工作区、版本库、暂存区

#主要命令

git add:添加到暂存区

git add . :add所有修改的文件

git commit -m” “:添加到提交

git status:查看状态

git diff <文件名> :查看文件修改

git log:查看提交历史

git log –pretty=oneline:简单地显示提交历史

git reflog:查看命令历史

git reset –hard HEAD^:返回上一个版本

git diff HEAD – readme.txt：查看工作区和版本库里面最新版本的区别：

git checkout – readme.txt：把readme.txt文件在工作区的修改全部撤销.修改包括新增、删除、文件修改

git reset HEAD 把暂存区的修改撤销掉（unstage），重新放回工作区

git rm

git clone git@github.com:objyg/git-test.git

#分支

git branch dev :创建分支(dev是你要切换的分支名)

git checkout dev:切换分支

git checkout -b dev:创建并切换分支

git switch -c dev:2.23版本创建并切换分支

git switch :新版本切换分支

git branch：查看当前分支

git branch -a ：查看远程分支

git merge dev 把dev分支的工作成果合并到master分支上

git branch -d dev:删除dev分支

git merge <分支>:把分支合并到主分支上
